# 贡献指南
感谢你愿意为本项目做出贡献！请阅读以下指南以帮助我们更高效地协作。

## 开发环境
- 本项目为纯静态站点，无需构建，直接打开 `index.html` 即可
- 可选：使用本地静态服务器（`python -m http.server` 或 `npx serve .`）

## 提交 Issue
- 使用 Issue 模板（缺陷/功能需求）
- 说明问题背景、复现步骤、期望结果与实际结果
- 附带环境信息（浏览器、操作系统）与截图/配置（如适用）

## 提交 PR
- 从 `main` 拉分支：`feature/...` 或 `fix/...`
- 保持 PR 小而聚焦；关联 Issue；在 PR 描述中列出变更点与测试方式
- 遵循项目结构与代码风格；不引入未必要的依赖

## 代码与文档
- 不在中部插入 `import`（保持文件顶部）
- 文档使用简体中文为主，必要处可附英文名词

## 发布与变更
- 重要变更请更新 `CHANGELOG.md`

## 行为准则
- 参与者需遵守 `CODE_OF_CONDUCT.md`
